# Open Graph Meta Tags — Social Media Preview That Works

You’ve spent time writing a great article or crafting a product page — and then someone shares it on social media… only to see a **blank card**, missing image, or wrong description.

To fix that, you need **Open Graph meta tags**.

Originally created by Facebook, Open Graph (OG) tags define how content appears when shared on:
- Facebook
- LinkedIn
- Twitter (via Twitter Cards)
- Discord, Slack, and many more

They don’t improve your search rankings, but they **supercharge your visibility** and **click-throughs** on social.

---

## Why OG Tags Matter

- Control your **preview title and description**
- Set a **thumbnail image**
- Customize previews per page or content type
- Avoid ugly, automatic guesses from crawlers

---

## Core OG Tags

```html
<meta property="og:title" content="The Ultimate Guide to Frontend Performance" />
<meta property="og:description" content="Learn how to make your apps faster with real-world patterns and tools." />
<meta property="og:image" content="https://example.com/preview.jpg" />
<meta property="og:url" content="https://example.com/performance-guide" />
<meta property="og:type" content="article" />
```

---

## Twitter Cards

Twitter uses its own meta format (but supports OG fallback):

```html
<meta name="twitter:card" content="summary_large_image" />
<meta name="twitter:title" content="The Ultimate Guide to Frontend Performance" />
<meta name="twitter:description" content="Speed up your apps with proven techniques." />
<meta name="twitter:image" content="https://example.com/preview.jpg" />
```

---

## Recommended Image Sizes

| Platform  | Type               | Suggested Dimensions     |
|-----------|--------------------|--------------------------|
| Facebook  | `og:image`         | 1200 × 630 px            |
| Twitter   | `summary_large_image` | 1200 × 675 px        |
| LinkedIn  | Follows OG tags    | 1200 × 627 px            |
| Discord   | Follows OG         | 1280 × 720 px            |

Always use HTTPS and static image URLs. Use JPG or PNG.

---

## Dynamic OG Tags in SPAs

In React, Next.js, Nuxt, etc., use server-side rendering or head management libraries:

### Next.js with Head

```tsx
import Head from 'next/head';

<Head>
  <meta property="og:title" content="Blog Post Title" />
  <meta property="og:image" content="/og-image.jpg" />
</Head>
```

---

## Anti-Patterns

- Using low-res or missing `og:image`
- Skipping `og:title` or `og:description`
- Using client-side rendering with no SSR fallback
- Reusing the same image across every page

---

## Conclusion: Own the Preview

Social media sharing is a front door to your content.

OG meta tags let you **design the welcome mat** — making your pages more clickable, recognizable, and shareable.

Because first impressions matter — and this one’s **in the `<head>`**.
